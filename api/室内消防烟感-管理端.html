# 1.1 首页地图接口(消防、烟感、水压 点位置及状态)

  • 说明
  • 请求URL

/managerFront/getIndexMapData

  • 请求方式

GET

  • 请求参数
请求参数 必选 参数类型 说明
equipmentType true Integer 设备类型 筛选条件 1为消防栓设备 2为烟感器设备 3为水压设备
  • 返回字段
返回字段 字段类型 说明
equipmentName String 消防栓名称
equipmentNumber String 消防栓编号
longitude String 经度
latitude String 纬度
equipmentStatus Integer 后端约定消防状态值 0为离线 1为正常 2为用水 3为开盖 4为水压低 5为电量低
后端约定烟感状态值 0为离线 1为正常 2为电量低 3为火警预警
后端约定水压状态值 0为离线 1为正常 2为电量低 3为水压低
  • 返回示例
// 正确示例
{
    "content": "",
    "code": "0000",
    "result": {
        "equipmentlist": [
            {
                "fireNumber": "0934000883784546",
                "fireName": "ZXH_XFM_I_TEST003",
                "latitude": "34.454554",
                "id": 152,
                "longitude": "117.23423",
                "equipmentStatus": 3
            }
        ]
    },
    "url": null
}
// 错误示例
{
    "content": "设备类型不能为空,请检查后重试!",
    "code": "0001",
    "result": null,
    "url": ""
}

imei String 设备IMEI
address String 设备地址

# 1.4 查看追踪报警记录

  • 说明
  • 请求URL

/managerFront/getAlarmlist

  • 请求方式

GET

  • 请求参数
请求参数 必选 参数类型 说明
equipmentType true Integer 设备类型 后台约定 1为消防栓设备 2为烟感器设备 3为水压设备
pageNum true Integer 当前页 即第几页
pageSize true Integer 页大小 即每页显示条数
startTime false String 开始时间 筛选条件
endTime false String 结束时间 筛选条件
alarmStatus false Integer 处理状态 筛选条件 1未处理 3已处理
alarmReason false String 报警原因 筛选条件 110=用水(异常)；120=开盖(异常)；130=电池电压低(故障)；140=水压低(异常);200=离线(故障);210=火警(异常)
fireKind false Integer 消火栓类型 筛选条件 0室内,1室外 (只有设备类型是消火栓时才传递)
  • 返回字段
返回字段 字段类型 说明
alarmlist List 报警事件list
total Integer 总条数(查询到的数据总条数)
current Integer 当前页(和传参时的pageNum一致)
size Integer 页大小(和传参时的pageSize一致)
pages Integer 总页数
address String 地址
personName String 维修人姓名
person Integer 维修人ID
lastWdate String 报警时间
faId Integer 报警事件ID
fireKind Integer 消火栓类型 0室内,1室外(只有设备类型为消防时才有该字段)
alarmReason String 报警原因 110=用水(异常)；120=开盖(异常)；130=电池电压低(故障)；140=水压低(异常);200=离线(故障);210=火警(异常)
deviceId Integer 设备ID
